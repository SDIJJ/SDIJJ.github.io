SpringCloud微服务学习总结

开源框架
  • java

骗的了别人骗不了自己,做什么都要用心!
对SpringCloud学习做一个总结,供以后应用查看

微服务的特点:

  • 一系列微小的服务共同组成
  • 单独部署,跑在自己的进程里
  • 每个服务为独立的业务开发
  • 分布式的管理

传统vs微服务

业务形态上不适合微服务的场景

  • 系统中包含很多很多强事务的场景
  • 业务相对稳定,迭代周期长
  • 访问压力不大,可用性要求不高
  • …

### 1.5小结

eureka服务注册中心在生产环境中可以做到高可用,启动多个eureka服务注册中心,彼此相互注册,每个客户端服务再分别注册到这些eureka服务注册中心.即使一个eureka垮掉,整个微服务也可以正常运行.

7.服务容错Hystrix

7,1Hystrix的功能

  • 服务降级
  • 依赖隔离
  • 服务熔断
  • 监控(Hystrix Dashboard)

7.2服务降级

优先核心服务,非核心服务不可用或若可用

通过HystrixCommand注解指定

fallbackMethod(回退函数)中具体实现降级逻辑

7,3Hystrix的使用

在client服务去访问client2服务的getMessage方法

  • 在client服务端添加Hystrix依赖

  • 启动类上添加@EnableCircuitBreaker注解

* 定义HystrixController实验服务降级

  >在方法上添加  @HystrixCommand(fallbackMethod = "fallback")注解当方法出现异常或者超时会自动调用fallback();
  >
  >如果想所有的方法都实现服务降级,可以在Controller上添加@DefaultProperties(defaultFallback="xxxxx") xxxxx为自定义服务降级走的方法

7,4Hystrix服务的熔断

断路器实现服务熔断:

访问getMessage方法,当参数为奇数是抛异常,当异常达到一定次数是(60%)启动断路开关
